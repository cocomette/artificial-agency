## Findings

1. **Addressed: required Kaggle metadata templates have been restored owner-neutral.**
   The public staging repo now includes the metadata inputs required by `kaggle/Makefile`, `kaggle/scripts/sync_kaggle_metadata.py`, `kaggle/scripts/build_notebook.py`, and `kaggle/scripts/build_debug_notebook.py`. The files use owner-neutral placeholders and `kaggle/.env.example` documents `FACE_OF_AGI_KAGGLE_OWNER` plus `FACE_OF_AGI_KAGGLE_TOKEN_FILE`; `make sync-metadata` stamps kernel, dataset, and model owner fields from `kaggle/.env`. `FACE_OF_AGI_KAGGLE_OWNER=kaggle-owner make notebook` was verified to generate the ignored submission notebook without upload credentials.

2. **Addressed: root license and third-party boundary files were added.**
   The staging repo now includes `LICENSE`, `LICENSE-CC-BY-4.0`, `LICENSE-APACHE-2.0`, `NOTICE`, and `THIRD_PARTY_LICENSES.md`. `pyproject.toml` declares `CC-BY-4.0 OR Apache-2.0`. The files state that covered project material is dual-licensed while third-party dependencies, model weights, datasets, and other external artifacts remain under their own terms.

3. **Addressed: local runtime state was pruned and ignored.**
   `resources/shared_memory.sqlite` and the generated `resources/` directory were removed. `.gitignore` now blocks SQLite/database outputs and generated runtime resource state.

4. **Addressed: private model import notebook workflow was removed.**
   The public repo does not need the helper notebook that imported Hugging Face weights into Kaggle storage. The submitted competition notebook is generated by `kaggle/scripts/build_notebook.py`; the removed bootstrap workflow was separate private infrastructure and is no longer part of the public staging tree.

5. **Addressed: Kaggle notebook reproduction is documented in `kaggle/README.md`.**
   The root repo no longer carries a dedicated Kaggle reproduction document. `kaggle/README.md` documents the clean-checkout path to `kaggle/notebooks/submission.ipynb`, the default submission config, required Kaggle inputs, and the fact that `kaggle/scripts/build_notebook.py` embeds only `src/face_of_agi` and `pyproject.toml`.

6. **Addressed: stale hard-coded Kaggle owner references were removed from `kaggle/README.md`.**
   The public Kaggle setup docs now point users at `kaggle/.env.example` and describe metadata syncing through `FACE_OF_AGI_KAGGLE_OWNER`. Stale Gemma and private model-upload Make targets were removed.

7. **Addressed: old project docs no longer recommend stale license alternatives.**
   `doc/project/arc-agi-3_concept.md` and `doc/project/arc-agi-3_technicals.md` now reflect the chosen `CC-BY-4.0 OR Apache-2.0` policy for covered project material and separate third-party dependencies, model weights, datasets, and external platform inputs.

8. **Partly addressed: third-party dependency boundaries are documented.**
   `THIRD_PARTY_LICENSES.md` now lists direct runtime, optional/test, and Kaggle notebook dependency surfaces. Before final publication, regenerate the wheelhouse and keep the exact downloaded package list, versions, license metadata, and notices with the release artifacts.

9. **Addressed: internal development-agent files were pruned.**
   `AGENTS.md` and `.agents/skills/**` were removed from the public staging tree.

10. **Addressed: internal work-package history was pruned.**
    `doc/work_packages/` was removed from the public staging tree. Stable architecture context remains under `doc/architecture/`.

11. **Addressed: generated/local catalogs and baseline JSON are pruned and ignored.**
    `src/face_of_agi/environment/local_games.json`, `debug/scoring/human_baseline.json`, and `debug/scoring/human_baseline_stats.json` are not tracked. README documents regenerating the local game catalog. The scoring dashboard now shows a missing-baseline warning instead of failing when the pruned stats file is absent.

12. **Addressed: unprovenanced binary architecture image was pruned.**
    `doc/architecture/framework.jpg` was removed. Architecture context remains in text and Mermaid-style docs.

13. **Addressed: private cloud upload/runtime paths were pruned.**
    Public Kaggle tooling now focuses on wheelhouse/public-game artifacts and generated submission/debug notebooks. Private model-upload scripts, dashboard snapshot pulling, and cloud runtime helpers have been removed from the staging tree.

14. **Addressed: `.gitignore` was extended for public safety.**
    Ignore rules now cover `.pytest_cache/`, runtime databases, parquet/arrow outputs, generated resources, generated Kaggle notebooks, local game catalogs, human-baseline JSON, and common model-weight/checkpoint file extensions.

15. **Addressed in README: default tests are documented as model-free.**
    The README states that the regression command does not call external model APIs or a live vLLM server. Kaggle and vLLM E2E paths remain manual workflows.
